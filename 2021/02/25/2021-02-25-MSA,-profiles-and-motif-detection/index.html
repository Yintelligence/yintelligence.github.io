MSA, profiles and motif detection

Video

Multiple sequence alignment? 25:42

An alignment of 2 sequences is an hypothesis that is supported or rejected by the score one obtains for the alignment, but…

  • the score does not guarantee that there really is a common ancestor
  • no guarantee that the elements are correctly aligned, only that the alignment is the one with maximum similarity

We can reduce this uncertainty by aligning additional sequences

A multiple sequence alignment (MSA) algorithm performs this task and can more clearly show:

  • The similarity between positions
  • The conservation of certain amino acids within a family of homologous proteins

Why MSA? 30:14

  • Improve the alignment between two sequences

  • Conserved regions provide information on the similarities in structure and function

  • Yet since the methods can only approximate the optimal solution, there can be differences between methods

Many MSA methods

The MSA problem 35:52

Determining the alignment between N sequences is a hard problem = Combinatorial optimisation problem (COP)

To solve this COP we need provide two systems:

  1. Calculating the score of the alignment
  2. Algorithm that finds the MSA with the optimal (or close to optimal) score

The scoring system 36:28

How can we calculate the score of an MSA?

  • The total score: S(m)=∑ S(mi)
    • So the assumption is made that the score of one column (i) is independent of the other columns
    • Sum-of-pairs or SP
  • The score of column i: S(mi)=∑ s(mki, mli)
    • mki is a residue in the sequence k in column i
    • s(mki, mli) The score from the substitution matrix

SP and its alternatives (column scores)

Minimum entropy: S(mi)=∑ fkiln(fki)

Maximum likelihood

Methods 3 40:20

Well-known methods:

  • An optimised version of the standard pairwise DP algorithms; limited applicability but exact
  • Progressive alignment (the CLUSTAL system)

  • Iterative improvement methods for multiple sequence alignment (the SAGA system)

Progressive alignment problems 45:25

This method suffers from at least 2 problems:

  • The local minimum problem
    • New sequences are added on top of existing alignments.
    • Hence any early errors (misaligned regions) propagate to the complete alignment (due to divergence between sequences)
  • The choice of alignment parameters
    • One scoring matrix, a gap-open and a gap-extension value
    • This works often for highly related sequences, yet starts to fail seriously as soon as sequences diverge

Iterative improvement 46:22

Profiles 57:04

Using a profile we can better detect similarities between distantly related proteins.

Profiles store general information properties of a MSA :

  1. Frequency information concerning the residues in each column of the MSA
  2. The evolutionary importance of every residue in that position

Position specific scoring matrix (PSSM)

How to construct a PSSM ? 57:56

Step 1: determine the frequencies for each amino acid b in every column u for Nseq aligned sequences

A profile stores for each column in the MSA, the frequency multiplied by the alignment score with every other AA.

The score mu,a is the score for aligning a residue a with the column u

  • We can therefore use the same DP algorithm as before to align a new sequence to an existing MSA (profile)

The biggest problem in creating profiles is that the number of sequences can be too small, resulting in qu,a of zero, and as a consequence not every amino acids is present in each column

  • log 0 =-∞

As a result the alignment can not be made using the log-formula (see PAM and BLOSUM discussion)

  • →PSEUDOCOUNTS

Once the profile is determined one can also derive a consensus sequence which contains for every column the amino acid with the highest score mu,a
